Running the GNN Training in Google Colab
=========================================

Workflow: edit locally -> git push -> run in Colab.
No Google Drive syncing required.

One-time setup:
  1. Make sure your local changes are pushed to GitHub:
       git add .
       git commit -m "your message"
       git push
  2. Open colab.ipynb directly in Colab:
       Go to https://colab.research.google.com -> File -> Open notebook
       -> GitHub tab -> search YosemiteSamurai/CurrentPrediction

Each session:
  Cell 1 -- Clones the repo from GitHub into /content/CurrentPrediction.
            If the repo is already cloned (e.g. runtime not restarted),
            it runs git pull to pick up any new commits instead.

  Cell 2 -- Adds gcn/ to sys.path so all imports work.

  Cell 3 -- Installs torch_geometric and wandb.
            (PyTorch is pre-installed on Colab, so this is fast.)

  Cell 4 -- Logs in to Weights & Biases. You will be prompted for your API key.
            Find it at: https://wandb.ai/settings

  Cell 5 -- Runs the hyperparameter sweep. This imports sweep.py, which
            automatically registers the sweep with W&B and starts the agent.
            The dataset is fetched from the OSU server inside dataset.py --
            no manual data download needed.

Note:
  sweep.py triggers wandb.sweep() and wandb.agent() at import time, so the
  sweep starts as soon as Cell 5 runs. If you want to run a single training
  pass without a full sweep, comment out the wandb.agent() line in sweep.py
  and call sweep.main() directly from the cell instead.
